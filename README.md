## Hyperlocal Delivery Intelligence Platform (TEAM-FLUX)

This repository contains an end-to-end **R + Power BI** project that implements the abstract in `abstract.pdf`:

- **Hyperlocal delivery analytics**
- **Star-schema data warehouse**
- **ETL in R**
- **Data mining and machine learning**
- **Interactive Power BI dashboards**

The project is designed to be suitable for a **20-mark academic assignment**.

---

### 1. Project Structure

- **`data/`**
  - `supply_chain.csv` – raw operational delivery data
  - `agent_reviews.csv` – raw customer/agent review data
  - `hyperlocal_dw.sqlite` – SQLite data warehouse (generated by R)
- **`etl/`**
  - `etl_supply_chain.R` – cleaning and transforming supply chain data
  - `etl_agent_reviews.R` – cleaning and transforming review data
  - `build_data_warehouse.R` – building the star schema and loading SQLite
- **`analytics/`**
  - `olap_analysis.R` – OLAP-style summaries and plots
  - `delay_prediction.R` – classification model for delay prediction
  - `clustering_analysis.R` – clustering agents/cities
  - `association_rules.R` – association-rule mining
  - `anomaly_detection.R` – outlier detection on deliveries
  - `text_mining_reviews.R` – text mining on review text
- **`R/`**
  - `install_packages.R` – helper script to install all required packages
- **`powerbi/`**
  - `README.md` – instructions and suggested dashboards
- **`report/`**
  - Your final report (Word / PDF) and exported plots/screenshots

You should place your real CSV files into the `data/` folder with the expected names.

### 2. Expected Columns in the Datasets

You can adjust the scripts if your column names differ. The code assumes the following typical structure:

- **`supply_chain.csv`**
  - `delivery_id` – unique id per delivery
  - `order_timestamp` – date-time of order (e.g., `2024-06-01 12:34:00`)
  - `delivery_time_min` – actual delivery time in minutes
  - `sla_min` – promised SLA time in minutes
  - `distance_km` – delivery distance
  - `city` – city name
  - `traffic_level` – values like `Low`, `Medium`, `High`
  - `weather` – values like `Clear`, `Rainy`, `Storm`, etc.
  - `vehicle_type` – e.g., `Bike`, `Scooter`, `Car`
  - `festival` – `Yes` / `No`
  - `restaurant_id` – restaurant/store identifier
  - `agent_id` – delivery agent identifier

- **`agent_reviews.csv`**
  - `review_id` – unique id per review
  - `order_id` – related order id (optional)
  - `agent_id` – delivery agent identifier
  - `agent_name` – delivery agent name
  - `rating` – numeric rating (e.g., 1–5)
  - `review_text` – free-text review
  - `order_accuracy` – numeric score (e.g., 1–5)
  - `product_availability` – numeric score (e.g., 1–5)
  - `discount_applied` – 0/1 or Yes/No
  - `price_range` – category like `Low`, `Medium`, `High`
  - `city` – city name
  - `order_date` – date of the order (date only)

If your column names are different, you can either rename the columns in R or adjust the scripts.

### 3. Tech Stack

- **Data Processing & ETL**: `R`, `tidyverse`, `dplyr`, `tidyr`
- **Data Warehouse**: `SQLite` via `DBI` and `RSQLite`
- **Data Mining & ML**: `caret`, `e1071`, `arules`, `cluster`
- **Text Mining**: `tidytext`, `tm`
- **Visualization in R**: `ggplot2`, `plotly`
- **BI & OLAP**: **Power BI** (connects to `hyperlocal_dw.sqlite`)
- **Version Control**: GitHub (optional but recommended)
- **Documentation**: Word / Markdown / LaTeX

### 4. How to Run

1. **Install R packages**
   - Open R or RStudio in this folder.
   - Run:
     - `source("R/install_packages.R")`

2. **Place raw data**
   - Put `supply_chain.csv` and `agent_reviews.csv` into the `data/` directory.

3. **Run ETL**
   - Run:
     - `source("etl/etl_supply_chain.R")`
     - `source("etl/etl_agent_reviews.R")`
     - `source("etl/build_data_warehouse.R")`
   - This will generate `data/hyperlocal_dw.sqlite`.

4. **Run analytics scripts**
   - Run any of the scripts in `analytics/`:
     - `source("analytics/olap_analysis.R")`
     - `source("analytics/delay_prediction.R")`
     - `source("analytics/clustering_analysis.R")`
     - `source("analytics/association_rules.R")`
     - `source("analytics/anomaly_detection.R")`
     - `source("analytics/text_mining_reviews.R")`

5. **Build Power BI dashboards**
   - Open Power BI Desktop.
   - Connect to `data/hyperlocal_dw.sqlite`.
   - Follow `powerbi/README.md` for suggested visuals and pages.
